Beginner's Guide

Today you'll learn how to:

  • Create and use a repo
  • Use issues
  • Configurate your repository

What is Git Center?

Git Center is a Git repository hosting and a collaboration system. Using git while completing this tutorial will be also helpful.

Creating a repo

Git Center repositories are just sites. To create a repo, press New repository on the home page and accept cloning.

After cloning you will be redirected back to Git Center and installation progress will begin. It can ask you to log in to ZeroID if you haven't done it yet. Installation usually doesn't take much time.

Finally you will be redirected to your repo root on Git Center zite. (Remember to give others http://127.0.0.1/your_repo_address instead of http://127.0.0.1/1GitLiXB6t5r8vuU2zC6a8GYj9ME6HMQ4t/repo/?your_repo_address because that adds more flexibility and makes changing Git Center source code easier.)

git clone

To clone your repo, simply copy git clone ... link from repository home page (click Clone button) and replace $path_to_data with ZeroNet's data directory. This works like cloning a local repo.

Fetching and publishing changes

  • git fetch
    There is nothing hard in fetching Git Center repositories. You can simply run git fetch.
  • git push
    Git Center offers two ways of pushing. First, you can run git push and then press Sign & Publish changes on repository home page.

    Secondly, you can skip manually pressing button with hooks. If you enable hooks on repository Settings tab, Git Center will fetch repository for changes and sign & publish repository for you.
    1. Your user must have read/write permissions to access ZeroNet directory.
    2. zeronet.home git variable must be set to absolute path.
    3. You must have python installed and available in PATH.
    You can disable signing and publishing via git push -o no-sign, publishing via git push -o no-publish and fetching via git push -o no-fetch. Join these arguments by using git push -o no-sign -o no-publish -o no-fetch.

Indexation

Right now you can share your repository by giving your friend a link to it. Git Center offers another way. You can index your repo.

  • First, open Settings tab.
  • You'll see Publish to Repository Index. You can press it to publish the repository. Note that you cannot completely remove your repository from index after you have published it.

Communication

To let people speak about some project, Git Center added support for issues. You can see them on Issues tab, create them by pressing New issue and then comment it. Other people can post comments then.

You can also have pull requests. If people fork your repository and then create a pull request with a reference to that fork you can accept these requests and merge them. To do that, press Import, then run git fetch on your local repository, merge branch pr-...-... manually and then git push.
